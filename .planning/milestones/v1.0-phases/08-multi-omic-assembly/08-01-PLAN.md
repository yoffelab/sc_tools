    **3. Create `sc_tools/assembly/_metadata.py`:**
    - `join_subject_metadata(modalities: dict[str, AnnData], *, subject_key: str = "subject_id") -> tuple[pd.DataFrame, pd.DataFrame]`
      - Validates subject_key exists in each modality obs (raise SCToolsDataError if missing per D-01)
      - Calls `validate_subject_metadata()` on each modality for consistency checks
      - Builds patient_metadata: outer join of unique subject_ids across modalities, with columns indicating which modalities are present (boolean flags)
      - Builds sample_metadata: concatenation of per-modality sample-level info (sample_id, subject_id, modality name)
      - Warns if zero subject_id overlap across any pair of modalities (Pitfall 4)
      - Returns (patient_metadata_df, sample_metadata_df)

    **4. Create `sc_tools/assembly/_build.py`:**
    - `build_mudata(modalities: dict[str, AnnData], *, subject_key: str = "subject_id") -> MuData`
      - Lazy import: `import mudata` inside function body (per CLI-08)
      - Calls `join_subject_metadata()` to validate and build metadata tables
      - Creates `MuData(modalities)` -- keys are modality names per D-04
      - Calls `mdata.update()` to sync global obs (Pitfall 1)
      - Stores patient_metadata and sample_metadata in `mdata.uns['patient_metadata']` and `mdata.uns['sample_metadata']` as DataFrames (per research recommendation)
      - Feature spaces remain separate per D-05 (no var merging)
      - Returns mdata

    **5. Create `sc_tools/assembly/_atlas.py`:**
    - `class MultiOmicAtlas` per D-06:
      - `__init__(self, mdata: MuData, metadata: dict[str, pd.DataFrame] | None = None)` -- wraps MuData, extracts metadata from uns if not provided
      - `@classmethod from_modalities(cls, modalities: dict[str, AnnData], *, subject_key: str = "subject_id") -> MultiOmicAtlas` -- calls `build_mudata()`, creates atlas
      - `@property patient_metadata -> pd.DataFrame` -- reads from `mdata.uns['patient_metadata']`
      - `@property sample_metadata -> pd.DataFrame` -- reads from `mdata.uns['sample_metadata']`
      - `patient_view(self, patient_id: str) -> MuData` -- subsets mdata to cells with `subject_id == patient_id` across all modalities per D-02
      - `sample_view(self, sample_id: str) -> MuData` -- subsets to sample_id
      - `celltype_proportions(self, *, celltype_key: str = "celltype", group_by: str = "subject_id") -> pd.DataFrame` -- delegates to `_query.celltype_proportions()`
      - `save(self, path: str | Path) -> None` -- `self.mdata.write(path)` (h5mu format per research recommendation)
      - `@classmethod load(cls, path: str | Path) -> MultiOmicAtlas` -- lazy `import mudata; mdata = mudata.read(path); return cls(mdata)`
      - `@property modalities -> list[str]` -- `list(self.mdata.mod.keys())`
      - `@property n_obs -> int` -- total cells across modalities

    **6. Create `sc_tools/assembly/_query.py`:**
    - `celltype_proportions(mdata: MuData, *, celltype_key: str = "celltype", group_by: str = "subject_id") -> pd.DataFrame` per D-10
      - Iterates over `mdata.mod.items()`
      - Skips modalities where celltype_key not in obs.columns (Pitfall 6)
      - For each modality: groupby [group_by, celltype_key], count cells, compute proportions within each group
      - Returns DataFrame with columns: [group_by, celltype_key, "modality", "count", "proportion"]
    - `aggregate_by_level(mdata: MuData, *, group_cols: list[str], value_col: str = "celltype") -> pd.DataFrame` per D-03
      - N-level abstraction stacking: groups by arbitrary columns
      - Filters to available columns per modality (some may lack "organ", etc.)
      - Returns DataFrame with columns: [*group_cols, value_col, "modality", "count", "proportion"]
