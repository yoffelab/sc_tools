  <action>
    **1. Create sc_tools/provenance/trace.py:**

    Implement `trace_lineage(file_path: str | Path, project_dir: str | Path = ".") -> list[dict]` following the research code example from 05-RESEARCH.md:

    - BFS walk using a queue, starting from the given file.
    - Maintain `visited: set[str]` using resolved canonical paths for cycle detection.
    - For each file: try `read_sidecar()`, if None and file ends with `.h5ad`, try `_read_uns_provenance()`.
    - If no provenance found: append step with `{"file": str, "command": None, "timestamp": None, "note": "origin (no provenance)"}`.
    - If provenance found: append step with `{"file": str, "command": prov["command"], "timestamp": prov["timestamp"], "note": None}`, then queue all input file paths.
    - For missing input files (per D-10): (1) check adata.uns if h5ad, (2) call `_find_by_sha256(sha256, project_dir)` to search for relocated file, (3) if found, continue trace and set note to "input relocated", (4) if not found, mark as origin.
    - Reverse steps at end for chronological order (oldest first, per D-12).

    Implement `_read_uns_provenance(path: Path) -> dict | None`:
    - Lazy import h5py.
    - Open file with `h5py.File(path, "r")`.
    - Check for `"uns/sct_provenance"` key.
    - If found, decode bytes to string, parse JSON, return dict.
    - If not found or any error, return None.

    Implement `_find_by_sha256(sha256: str, project_dir: str | Path) -> Path | None`:
    - Walk project_dir recursively looking for files with matching SHA256.
    - Only check files that could plausibly be data files (skip hidden dirs, .git, __pycache__).
    - Return first match or None.
    - Use a reasonable limit (e.g., check at most 1000 files) to avoid scanning enormous directories.

    **2. Create sc_tools/tests/test_cli_provenance.py** with tests for trace_lineage:

    - Use `tmp_path` to create sidecar chains. For a 3-step chain test: create files A, B, C in tmp_path. Write sidecar for C referencing B as input. Write sidecar for B referencing A as input. A has no sidecar. Call `trace_lineage(C, project_dir=tmp_path)`. Verify 3 steps in chronological order (A first, C last).
    - For cycle detection: create sidecars where A references B and B references A. Verify trace completes without hanging, returns 2 steps.
    - For adata.uns fallback: create an h5ad file with `uns/sct_provenance` embedded as JSON string (use h5py directly). Call trace on it without a sidecar file. Verify provenance is read from uns.
    - For SHA256 relocation: write a file, compute its SHA256, write a sidecar referencing the file at a path that does not exist, move the file to a different path in project_dir. Call trace and verify the file is found by SHA256 scan.
