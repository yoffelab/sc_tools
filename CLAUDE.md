## Interaction Rules

### Work Mode (implementing, running code, making changes)
After EVERY prompt in work mode:
- Update project `Mission.md`: mark completed steps `[x]`, update "In Progress", add blockers/next steps

### Journal Workflow
- **`Journal.md`** (per project) — append dated entries after significant work (action, rationale, decisions)
- Run `make wiki-sync` after phase transitions to update `.gen.md` files

### Phase Transitions
When completing a phase, use the MCP tool to record counts:
```
set_phase_status(project, phase, "complete", n_obs=N, n_vars=N, n_samples=N, notes="...")
```
Then run `make wiki-sync` to regenerate `status.gen.md` with the new data.

### Suggestion Files
- Agent writes `.suggest.md` files (e.g. `hypotheses.suggest.md`) for human review
- Human promotes content to `.md` (accept) or deletes `.suggest.md` (reject)
- Never edit `.gen.md` files — they are auto-generated by `make wiki-sync`

### Plan Lifecycle
- Plans auto-save to `docs/wiki/plans/untiered/` on ExitPlanMode (via hook at `~/.claude/hooks/save_plan.py`)
- Plan files: `YYYY-MM-DD-{slug}.md` with YAML frontmatter (`status`, `created`, `summary`)
- Plan status: `active` → `in_progress` (set manually when starting) → `complete` (set when done)
- When starting a plan: update `status: in_progress` in the plan frontmatter
- When completing a plan: update `status: complete` + add Journal.md entry + `make wiki-sync`
- Mission.md `## Active Plans` is auto-generated by `make wiki-sync` (scans all plan tiers)
- **Always keep plans** — never delete; promote through versions (V1 → V1.1 → V2) by saving new files

### Plan Tiers
| Tier | Path | Scope |
|------|------|-------|
| **Tier 1** | `docs/wiki/plans/tier1/` | Large-scale architectural plans: overall data flow changes, pipeline restructuring, cross-project decisions |
| **Tier 2** | `docs/wiki/plans/tier2/` | Sub-task plans: single module, single project phase, feature implementation |
| **Untiered** | `docs/wiki/plans/untiered/` | Default auto-save destination; promote to tier1/tier2 manually |

**Versioning:** Append version suffix to slug: `YYYY-MM-DD-{slug}-v1.md`, `YYYY-MM-DD-{slug}-v1-1.md`, `YYYY-MM-DD-{slug}-v2.md`. Keep all versions — never overwrite.

Plan mode (discussing only): updating Mission optional until execution begins.

## Container / Runtime

**Priority: Apptainer first (auto via `scripts/run_container.sh`):**
- Linux/HPC → Apptainer (primary)
- macOS/Windows → Docker (Apptainer not native; fallback)

**Running scripts:**
```bash
./scripts/run_container.sh projects/visium/ggo_visium python scripts/foo.py
./scripts/run_container.sh projects/visium/ggo_visium   # interactive bash
```

**Override:** `SC_TOOLS_RUNTIME=docker|apptainer ./scripts/run_container.sh ...`

**Build SIF:** `docker build -t sc_tools:latest . && apptainer build containers/sc_tools.sif docker-daemon://sc_tools:latest`

**Snakemake:** `snakemake -d projects/visium/ggo_visium -s projects/visium/ggo_visium/Snakefile <target>`

`run_apptainer.sh` and `run_docker.sh` are now thin wrappers around `run_container.sh`.

## Key Conventions

- **Paths:** All outputs under `projects/<platform>/<project>/` — no root-level results/figures/metadata
- **Checkpoint names:** Standard names only (see docs/Architecture.md §2); no ad-hoc filenames
- **Script paths:** Use `$PROJECT_DIR` (set by run_container.sh) or Makefile `$(PROJECT)`
- **Signature scores:** `obsm['signature_score']` / `obsm['signature_score_z']`; not in `obs` by default
- **Colors:** `adata.uns[f'{name}_colors']`; never overwrite if length matches
- **No files > 1MB** in git
- **No apostrophes** in generated documentation text
- **Legacy:** `scripts/old_code/` read-only; refactor into sc_tools or new scripts
- **New project:** `./projects/create_project.sh <name> <data_type>`

## Analysis Standards

Full standards: @docs/skills.md. Quick reference:
- FDR: Benjamini–Hochberg always; report adjusted p-values
- Significance bars: `*` < 0.05, `**` < 0.01, `***` < 0.001, `****` < 0.0001
- Group comparisons: 2 groups = pairwise; >2 groups = 1-vs-rest (default), all-pairwise as option
- Deconvolution: batch per `library_id` to avoid OOM
- Linting: Ruff (`make lint`) — never commit failing lint
- **Publication figures** (docs/skills.md §12): 300+ DPI, Helvetica/Arial 5-8pt, color-blind safe palettes (Okabe-Ito default, no jet/rainbow), bold panel labels (lowercase for Nature, uppercase for Cell/Science), scale bars on spatial plots, exact P-values preferred. Use `marsilea` for complex composite figures.
