# BioData OOP Hierarchy v1.0 — JTI Type Hierarchy, Platform Registry, Subject/Sample Model

## Summary

Initial implementation of the BioData type hierarchy for sc_tools. Replaced the flat `datasets` table with a typed, extensible system supporting 5 data modalities.

## Deliverables (all complete)

1. **PlatformSpec registry** (`sc_tools/biodata.py`): 95 platforms across spatial_seq, image, rnaseq, epigenomics, genome_seq categories. Pure Python, no DB dependency.

2. **JTI ORM hierarchy** (`sc_tools/registry.py`):
   - `BioData` base table with shared columns (uri, format, status, n_obs, n_vars, phase, etc.)
   - `BioImage` — image-specific: n_channels, pixel_size, staining_protocol
   - `RNASeqData` — RNA-seq: library_type, chemistry, reference_genome
   - `SpatialSeqData` — spatial seq: spatial_resolution, panel_size, bin_size
   - `EpigenomicsData` — epigenomics: assay_type, target_protein, n_peaks
   - `GenomeSeqData` — genome seq: sequencing_type, panel_name, coverage

3. **Subject/Sample model**:
   - `subjects` table: cross-project de-identified patient records (sex, diagnosis, vital_status, clinical_metadata_json overflow)
   - `samples` table: physical specimens linking subjects to projects
   - `subject_project_links`: many-to-many join with role

4. **Registry API**: `register_biodata()`, `get_biodata()`, `list_biodata()`, `project_data_summary()`, `add_subject()`, `add_sample()`, `link_subject_to_project()`, `migrate_datasets_to_biodata()`

5. **MCP tools** (7 new): add_subject, list_subjects, add_sample, list_samples, register_biodata, list_biodata, project_data_summary

6. **Alembic migration** `0005_biodata_hierarchy.py`: creates all new tables + `bio_data_id` FK on datasets

7. **Seed script** (`scripts/seed_biodata.py`): populated 142 synthetic subjects, 948 samples, migrated 59 datasets to BioData
